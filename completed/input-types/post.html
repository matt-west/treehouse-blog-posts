Using HTML5 Input Types to Enhance The User Experience on Mobile

Browsing the web on mobile devices has become incredbily popular over the past few years. The problem is that the browsing experience on these devices can sometimes leave a lot to be desired. The Responsive Web Design movement started by Ethan Marcotte has helped to stress the importance of providing a fantastic user experience on mobile.

However, our focus shouldn't stop at layouts and navigation. One of the worst experiences on mobile is having to fill in forms. Luckily the HTML5 specification introduced a number of new input types you can use to specify what type of data you want to collect in your forms. In a rather awesome move, mobile browser vendors have picked up on these new input types and are using them to display customized keyboard layouts to users that help them to enter data quickly and easily.

In this blog post you are going to learn about eight new input types that you can utilise to make it easier for your users to fill in forms on their mobile devices.

Note: The iOS screenshots in this post where taken using an iPhone 5 and Safari. The Android screenshots were taken using a virtual device running Android 4.1 and the built-in web browser.

Email Addresses

The email type prompts both the iOS and Android browsers to display slightly customized keyboards. Notice the presence of a shortened space bar and addition of @ and . keys in the bottom row of the iOS keyboard. On Android, the standard comma key that would appear to the left of the space bar has been replaced with an @ key. 

<input type="email" name="email">

URLs

The url input type can be used to help users input web addresses. On iOS the whole space bar has been replaced with ., / and .com keys.

My testing showed no changes to the Android keyboard. 

<input type="url" name="url">

Telephone Numbers

Using the tel input type prompts both iOS and Android to display a dialer keypad instead of the standard keyboard. 

<input type="tel" name="tel">

Numbers

The number input type causes iOS to present a keyboard with numbers and punctuation; whereas Android will display a keypad similar to the one displayed for telephone inputs. 

<input type="number" name="number">

Date

There are also a number of types available for inputing dates and times. These can be really useful as it ensures that your data is provided in a standardized format.

On iOS the date input type will prompt a date picker to be displayed. Unfortunately Android does not yet have support for any datetime input types. 

<input type="date" name="date">

Time

Using the time type will prompt iOS to display a simple picker for selecting hours and minutes. 

<input type="time" name="time">

Date and Time

The datetime type will display a picker for selecting both a date and time.

It is worth noting that although the is no option for explicitly selecting a year the picker will automatically add a year to your input based on the date and month that you select. 

<input type="datetime" name="datetime">

Month

The month type will display cut-down version of the date picker.

There is also a week input type defined in the HTML5 specification however this does not seem to have been implemented on either iOS or Android. 

<input type="month" name="month">

A Note on Browser Compatibility

Browsers that do not support these new input types will just display simple text inputs to users. This means that you can go ahead and start using these new input types straight away!

Support for the date/time input types amongst desktop browsers is still very limited. Opera currently has the best implementation, supporting all of the types mentioned in this post. Google Chrome has support for the date type but nothing else at the moment. Safari has date-formatted text fields but does not support the calender widget that is displayed in Opera and Chrome.

Final Thoughts

Fortuantely the days of having to endure a sub-standard browsing experience on mobile are swiftly on the way out. By following the principles of Responsive Web Design and taking advantage of the new mobile-friendly features introduced in HTML5 we can ensure that we are delivering an enjoyable experience to everyone, regardless of the device they are using.
